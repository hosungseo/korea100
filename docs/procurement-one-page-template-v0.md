# 조달 제도 한 장 템플릿 v0

기준일: 2026-07-12

`docs/one-page-template.md`(9칸 캔버스)를 승계하고, 조달 도메인에 필요한 항목을 추가한다. 데이터 계약 반영 시 이 문서가 스키마 확장(v3)의 근거가 된다.

## 9칸 캔버스 (조달 관점 재해석)

| 칸 | 질문 | 조달 버전에서 특히 담을 것 |
| --- | --- | --- |
| 1 | 이 제도는 무엇을 해결하나 | 어떤 계약 상황에서 이 제도를 쓰는지, 안 쓰면 어떻게 되는지 |
| 2 | 누구에게 영향을 주나 | 조달업체·수요기관·계약공무원 중 누가 주체이고 누가 상대방인지 |
| 3 | 법적 근거는 무엇인가 | 법률→시행령→시행규칙→계약예규→고시의 위계를 층위별로 명시 |
| 4 | 누가 권한을 갖나 | 계약담당공무원, 계약심사·심의 기구, 조달청, 기재부·행안부 |
| 5 | 절차는 어떻게 흐르나 | 공고·마감·통보 등 기한이 걸린 단계를 빠짐없이 |
| 6 | 돈은 어떻게 흐르나 | 보증금, 수수료, 선금, 기성·준공대가, 지체상금, 과징금 |
| 7 | 문서와 데이터는 어떻게 흐르나 | 나라장터 화면 단위 행위(공고, 투찰, 낙찰통보, 전자계약)와 서면 문서 |
| 8 | 어디서 막히나 | 탈락·반려·유찰·분쟁이 실제로 발생하는 지점 |
| 9 | 어떻게 바꿀 수 있나 | 개선 논의, 국가·지방 기준 불일치, 비교 제도 |

## 조달 특화 추가 항목

기존 스키마에 없어서 확장이 필요한 항목. 파일럿 1호 제작에서 필드명을 확정한다.

### 이용자 태그 (`audience`)

- 값: `업체`, `수요기관`, `계약공무원` 중 1개 이상
- 용도: 홈 필터, 상세 상단 `이 장을 읽는 사람` 표시
- 이용자별 첫 행동 한 줄을 함께 적는다. 예: `업체 — 나라장터에서 입찰참가자격 등록을 먼저 마친다.`

### 적용 범위 (`scope`)

- 값: `국가계약`, `지방계약`, `공통`, `조달청 고유`
- 국가·지방의 기준이 다른 제도는 본문에서 차이를 표로 정리하거나 별건으로 분리한다.

### 금액·비율·기한 기준 (`thresholds`)

- 수의계약 한도, 낙찰하한율, 보증금률, 대금지급 기한 같은 수치 항목
- 각 수치는 `값 + 근거 조문 + 기준일`을 한 세트로 기록한다.
- 조문 대조를 통과하지 않은 수치는 게시하지 않는다. 자리에는 `검증 후 게재`를 표시한다.
- 국가·지방·조달청 기준이 다르면 각각 별도 행으로 적는다.

### 흔한 실수 (`pitfalls`, 선택)

- 신규 진입자가 자주 겪는 탈락·반려 사유. 예: 마감 시각 착오, 규격 미충족, 증빙 누락
- 법령으로 확인되지 않는 실무 관행은 `fieldVerification`으로 표시한다.

## 한 장 레이아웃

### 상단

- 제도명, 한 줄 요약, 제도 유형
- 이용자 태그와 적용 범위 배지
- 기준일, 근거 법령 (법률/예규/고시 층위 표시)

### 중앙 — 상태 인식형 swimlane

- 행위자 lane 기본형: `조달업체` / `수요기관` / `계약담당(조달청 또는 발주기관)` / `심사·심의기구` / `감독·분쟁기구`
  - 제도에 따라 lane을 줄이거나 바꾼다. 예: 지정형 제도는 `신청기업 / 조달청 / 심사위원회`
- 단계 게이트: 공고 전 → 공고·접수 → 심사·평가 → 계약 체결 → 이행 → 사후관리
- 필수 표기: 현재 공 보유자, 기한이 걸린 화살표(마감·통보 기한), 회귀 루프(보완 요구, 재입찰, 재심사), 병목
- 법령상 절차와 현장 추정은 색상/라벨로 구분한다.

### 하단

- 돈의 흐름 (보증금·수수료·대가의 타임라인)
- 문서/데이터 흐름 (나라장터 단계 포함)
- 금액·비율·기한 기준표
- 병목과 쟁점, 개혁 포인트
- 관련 제도 (여정 연결: 이 제도의 앞뒤에 오는 제도)

## 작성 규칙

기존 규칙(한 장 완결, 조문번호 필수, 법령/추정 구분, 짧은 문장)에 다음을 더한다.

1. 조문번호는 국가법령정보센터 대조를 통과하기 전에는 `검증 전` 상태로 두고 공개 빌드에 싣지 않는다.
2. 계약예규·고시를 인용할 때는 공식 제명과 행정규칙 ID를 함께 기록한다. 통칭(예: `적격심사기준`)만으로 끝내지 않는다.
3. 수치(금액·비율·기한)는 값·근거·기준일 세트 규칙을 지킨다.
4. 국가계약 기준으로 서술하고 지방계약이 다르면 그 자리에서 차이를 밝힌다. 차이가 3곳 이상이면 별건 분리를 검토한다.
5. 이용자 태그별로 `이 제도에서 당신이 할 일` 한 줄을 상단에 배치한다. 그림을 읽기 전에 자기 lane을 찾게 한다.
6. 나라장터 화면 단계는 법령이 아니라 시스템 사용 절차다. `fieldVerification` 또는 낮은 `confidence`로 구분하고, 조달청 공개 매뉴얼을 출처로 연결한다.
7. 벌칙·제재(부정당 제재, 보증금 귀속)가 걸린 단계는 swimlane에서 경고 표시를 붙인다.
