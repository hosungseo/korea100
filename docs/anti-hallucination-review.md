# 국가법령 MCP(korean-law-mcp) 환각 최소화 기법 검토 보고서

- 검토일: 2026-07-15 (운영자 지시)
- 검토 대상: `korean-law-mcp@4.7.2` 빌드 소스 (`build/lib`, `build/tools`, `tool-registry`)
- 목적: 패키지가 LLM 환각을 줄이기 위해 구현한 기법을 식별하고, 이 리포의 조달제도 콘텐츠 파이프라인에 이식·적용한다.

## 3. 이식 결과: 인용 '내용' 기계 대조

`web/scripts/verify-citation-content.mjs` — citation-content-matcher의 순수함수(정규화·bigram·LCS)를 이식하되, 원본의 Jaccard는 '짧은 제목끼리' 비교용이라 우리 케이스('짧은 요지' vs '긴 조문 원문')에서는 분모가 커져 전부 불일치로 나온다(첫 실행: 807건 중 577건 과검출). 이를 **containment**(요지 bigram이 원문에 포함되는 비율, 임계 0.5)로 보정했다.

- 판정: L1(연속 30자+ 일치) → L2(containment ≥ 0.5) → MISMATCH
- 대상: 각 노드 `legal_basis.text`(요지) ↔ `verification.articleTexts`(법제처 현행 원문)
- 결과 리포트: `docs/citation-content-report.json`
- 첫 실행(2026-07-15): **807건 대조, 불일치 후보 113건(14%)**

불일치 후보의 성격(표본 검토):
1. **진짜 오귀속 의심** — 예: contract-signing P02가 '낙찰통지 후 10일 이내 체결'을 시행규칙 제49조제1항에 귀속(원문과 겹침 0.18). 검증자 재확인 대상.
2. **원문 데이터 한계** — articleTexts가 1400자 절단이거나 항 미추출(항 추출 파서는 3919e2c에 반영, 데이터 재생성 대기)이라 요지가 가리키는 항이 원문 조각에 없음. 재생성 후 자동 해소 예상.
3. **요지 자체가 미확정 자백** — "조문 대조 필요" 등을 담은 요지. fieldVerification로 이관해야 할 대상.

**운영 방식**: 보고서 전용(report-only)으로 시작한다. 113건에는 데이터 한계(2)가 섞여 있어 즉시 하드 게이트로 쓰면 오탐이 배포를 막는다. 조문 원문 재생성(항 추출 반영) 후 잔존 불일치를 검증자가 소거하고, 그 다음 `--strict`(불일치 시 exit 1)를 CI 게이트로 승격한다.

## 4. 운영 규칙 반영

- agent-guide 8절(작성·검증 에이전트 공통):
  1. MCP/CLI 출력의 `[NOT_FOUND]`·`[NO_CITATIONS_FOUND]`·오류는 **실패**다. 추측으로 메꾸지 말고 실패를 보고하거나 제안된 재시도 질의로만 재시도한다.
  2. 초안 제출 전 자기 인용을 `legal_analysis(mode=verify_citations)` 또는 원문 대조로 셀프체크한다.
  3. 요지(legal_basis.text)는 원문에 실재하는 표현·수치로 쓴다 — 내용 대조(containment)에 걸리지 않을 수준으로 원문에 밀착시키고, 원문에서 확인 못 한 서술은 요지에 넣지 말고 fieldVerification로 보낸다.
- 검증 체인(최종): 실존 대조(`verify:articles`) + 내용 대조(`verify-citation-content`) + 원문 그대로 표시(articleTexts, 규칙 15) — 3층 모두 결정적(비-LLM).

## 5. 남은 과제

1. 조문 원문 재생성(항 추출 파서 반영) 후 내용 대조 재실행 → 데이터 한계발 불일치 소거.
2. 잔존 불일치(진짜 오귀속 후보)를 검증 에이전트에 배정해 요지 정정 또는 fieldVerification 이관.
3. 정리 완료 후 `verify-citation-content --strict`를 law-verify CI에 게이트로 추가.
4. 팝업에 '시행 예정 버전' 경고(기법 6) 노출 여부 검토.
