## 13. 요청·지표·검증 운영 보완

- 요청 폼은 설정된 JSON API가 2xx를 반환한 경우에만 저장 완료로 표시한다.
- 수집 주소가 없거나 실패하면 입력 초안을 브라우저에 보존하고 이메일과 공개 GitHub 이슈 경로를 제공한다.
- 검색어 원문과 폼 내용을 제외한 최소 제품 이벤트를 선택형 분석 엔드포인트로 보낼 수 있게 했다.
- 100개 제도의 `fieldVerification`에서 448개 작업을 생성하는 현장 검증 큐와 `/verification/` 공개 대장을 추가했다.
- 현장 검증 항목은 제도, 분야, 검증 영역, 권장 근거, 안정적인 ID를 갖는다.
- 공식 법령 원문의 버전 서명을 레지스트리와 비교하는 `check:freshness` 모드와 주간 GitHub Actions를 추가했다.
- 변경이 발견되면 `law-freshness` 이슈를 만들거나 갱신하며, 저장소 secret `LAW_OC`가 없으면 쓰기 전에 실패한다.

## 16. 디자인 엔지니어링·모션 보완

- `emilkowalski/skills`의 디자인 엔지니어링·Apple 인터랙션·모션 검토 원칙을 반복 사용형 정책 도구에 맞게 선별 적용했다.
- `ease-out`, `ease-in-out`, drawer 곡선과 120–240ms 범위의 공용 모션 토큰을 추가했다.
- 버튼과 주요 명령에 즉시 눌림 피드백을 추가하고 hover 효과는 정밀 포인터 환경으로 제한했다.
- 비교 도구, 비교 패널, 요청 경고처럼 간헐적으로 나타나는 상태에만 짧은 진입 전환을 적용했다.
- 업무구조도의 반복 pulse, `transition: all`, 진행률 폭 애니메이션을 제거하고 현재 상태는 정적인 테두리와 라벨로 표시했다.
- 노드 drawer는 오른쪽에서 들어온 경로로 닫히며, 포인터 종료는 160ms, Escape 종료는 즉시 처리한다.
- drawer가 열리면 닫기 버튼으로 포커스를 옮기고 내부에서 순환하며, 닫힌 뒤 원래 업무 노드로 복원한다.
- sticky 헤더와 필터에는 반투명 재질과 불투명 fallback을 적용했다.
- `prefers-reduced-motion`, `prefers-reduced-transparency`, `prefers-contrast` 대응을 분리했다.
- 모바일 필터는 40px, 상세 보조 도구는 36px, drawer 닫기 버튼은 40px 터치 영역으로 조정했다.
