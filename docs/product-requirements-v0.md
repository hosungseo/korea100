# 제품 요구사항 문서 v0

> 이 문서는 첫 10개 공개를 정의한 역사적 MVP 기록이다. 현재 100개 공개 서비스의 요구사항은 `docs/product-requirements-v1.md`를 따른다.

## 제품명

한 장으로 끝내는 대한민국 제도 100

## 목적

대한민국의 주요 제도를 법령, 조직, 절차, 예산, 문서, 데이터, 권한 관계까지 한 장의 구조도로 보여주는 온라인 제도 모델 서비스와 오프라인 책을 만든다.

1차 제품은 홈페이지다. 홈페이지에서 첫 10개 제도 샘플을 공개하고, 독자 반응과 검색 수요를 검증한 뒤, 검증된 포맷을 오프라인 책으로 확장한다.

## 배경

한국의 제도는 법률, 대통령령, 시행규칙, 고시, 부처 직제, 예산사업, 신청 서식, 정보시스템이 결합되어 작동한다. 하지만 독자 대부분은 제도가 실제로 어떤 기관과 문서, 예산, 절차를 통해 움직이는지 한눈에 보기 어렵다.

이 제품은 제도를 텍스트 해설이 아니라 `제도 모델`로 보여준다. 기업에 비즈니스 모델이 있듯이, 국가에는 제도 모델이 있다는 관점이 핵심이다.

## 대상 사용자

- 정책 이슈를 빠르게 구조화해서 이해하고 싶은 일반 독자
- 신규 공무원, 공시생, 행정학/정책학 학생
- 국회ㆍ지방의회 보좌진과 정책 담당자
- 정책연구자, 컨설턴트, 공공기관 실무자
- 기자, 시민단체, 로스쿨 학생

## 사용자 문제

- 제도 설명이 법령 조문, 보도자료, 해설 글로 흩어져 있어 전체 구조를 보기 어렵다.
- 어느 기관이 결정하고, 어느 기관이 집행하고, 어떤 문서와 예산이 오가는지 파악하기 어렵다.
- 법령상 절차와 실제 업무 흐름의 차이를 구분하기 어렵다.
- 비슷한 제도를 같은 틀로 비교하기 어렵다.
- 책으로 바로 내기 전에 실제 독자 수요와 반복 가능한 제작 포맷을 검증할 필요가 있다.

## 제품 가설

첫 10개 제도를 한 장 구조도와 상태 인식형 업무구조도로 공개하면, 독자는 제도 이해의 효용을 즉시 느끼고 다음 제도 제작 요청 또는 출간 알림으로 반응할 것이다.

## 성공 기준

- 첫 10개 제도 샘플이 같은 포맷으로 공개된다.
- 환경영향평가 샘플은 상태 인식형 업무구조도까지 보여준다.
- 독자는 제도별로 근거 법령, 핵심 행위자, 절차, 산출물, 병목을 한 화면에서 파악할 수 있다.
- 홈페이지에서 다음 제도 제작 요청을 받을 수 있다.
- 출간 알림 또는 문의를 받을 수 있다.
- 방문, 체류, 요청, 공유, 피드백 데이터를 오프라인 책 기획 근거로 사용할 수 있다.

## MVP 범위

### 포함

- 홈페이지 첫 화면
- 제도 모델 설명
- 첫 10개 제도 목록
- 환경영향평가 대표 샘플
- 제도별 상세 패널 또는 상세 페이지의 기본 구조
- 상태 인식형 업무구조도 샘플
- 다음 제도 제작 요청 폼
- 출간 알림 또는 문의 수집 장치
- 법령 기준일 표시
- 모바일 대응

### 제외

- 회원가입과 로그인
- 결제
- 관리자 페이지
- 완전 자동 법령 분석
- 100개 전체 제도 완성
- 법률 자문 또는 공식 해석 제공

## 첫 공개 제도 10개

1. 환경영향평가
2. 예비타당성조사
3. 재건축ㆍ재개발 정비사업
4. 지방소멸대응기금
5. 정보공개청구
6. 행정심판
7. 국민기초생활보장
8. 공공조달ㆍ국가계약
9. 국민건강보험
10. 개인정보 영향평가

## 핵심 기능 요구사항

### 1. 홈페이지 첫 화면

- 제목은 `한 장으로 끝내는 대한민국 제도 100`을 사용한다.
- 핵심 문장은 `기업에는 비즈니스 모델이 있듯이, 국가에는 제도 모델이 있다.`를 사용한다.
- 첫 화면에서 환경영향평가 샘플 또는 그 일부가 보여야 한다.
- 사용자는 첫 10개 제도 목록과 제작 요청 흐름으로 이동할 수 있어야 한다.

### 2. 제도 모델 목록

- 첫 10개 제도를 우선순위와 유형별로 보여준다.
- 각 제도는 한 줄 설명, 유형, 왜 먼저 만드는지를 포함한다.
- 선택한 제도는 상세 패널 또는 상세 페이지에서 더 구체적인 내용을 보여준다.

### 3. 제도 상세 구조

각 제도 상세는 다음 항목을 담는다.

- 제도명
- 한 줄 요약
- 제도의 목적
- 근거 법령
- 대상자, 수혜자, 규제대상
- 핵심 행위자와 권한
- 절차 swimlane
- 돈의 흐름
- 문서/데이터 흐름
- 병목과 쟁점
- 개혁 포인트
- 관련 제도
- 법령 기준일

### 4. 상태 인식형 업무구조도

- 절차형 제도는 단계 게이트를 보여준다.
- 각 단계에서 현재 업무 보유자, 산출물, 다음 게이트, 병목, 보완 회귀 루프를 표시한다.
- 법령상 구조와 현장 검증 필요 지점을 구분한다.
- 환경영향평가를 첫 기준 샘플로 사용한다.

### 5. 다음 제도 제작 요청

폼은 다음 항목을 받는다.

- 알고 싶은 제도명
- 왜 궁금한지
- 독자 유형
- 가장 헷갈리는 지점
- 이메일 또는 연락처

요청 데이터는 이후 온라인 연재 순서와 오프라인 책 목차 우선순위를 정하는 근거로 사용한다.

### 6. 출간 알림과 문의

- 사용자는 오프라인 책 출간 알림을 신청할 수 있어야 한다.
- 문의 또는 피드백을 보낼 수 있어야 한다.
- 초기 MVP에서는 외부 폼, 이메일 링크, 또는 localStorage 기반 임시 저장도 허용한다.

## 콘텐츠 요구사항

- 모든 제도는 같은 반복 포맷을 따른다.
- 법률, 대통령령, 시행규칙을 우선 근거로 삼는다.
- 법제처 API로 현행 법령 근거를 추적한다.
- Mermaid/BPMN/HTML 업무구조도 산출물을 함께 활용한다.
- 법령상 절차와 실제 운영 추정은 명확히 구분한다.
- 근거가 약한 설명은 `현장 검증 필요`로 표시한다.

## 비기능 요구사항

- 정적 홈페이지로 먼저 운영할 수 있어야 한다.
- 모바일에서 가로 넘침 없이 읽을 수 있어야 한다.
- 제도 구조도는 이미지 또는 SVG로 공유 가능해야 한다.
- 검색 유입을 위해 제도명, 절차명, 법령명을 텍스트로 노출한다.
- 새 제도를 추가할 때 기존 포맷을 반복할 수 있어야 한다.

## 검증 지표

- 방문자 수
- 제도별 페이지 체류 시간
- 샘플 공유 수
- 다음 제도 제작 요청 수
- 직접 문의 수
- 반복 요청되는 제도명
- 피드백에서 반복되는 혼란 지점
- 공무원, 보좌진, 기자, 연구자, 학생 등 핵심 독자군의 반응

## 출시 단계

### 0단계: 현재 MVP

- 정적 홈페이지
- 첫 10개 제도 목록
- 환경영향평가 샘플
- 제작 요청 폼 초안

### 1단계: 공개 샘플 10개

- 첫 10개 제도에 대해 같은 포맷의 한 장 모델을 만든다.
- 환경영향평가, 예비타당성조사, 재건축ㆍ재개발 정비사업을 우선 완성한다.

### 2단계: 온라인 검증

- 홈페이지, 브런치, 링크드인, 정책 커뮤니티에 공개한다.
- 독자 반응과 제작 요청을 모은다.
- 템플릿과 설명 밀도를 조정한다.

### 3단계: 오프라인 책 전환

- 검증된 제도 모델을 100개까지 확장한다.
- 분야별 색인, 제도 유형별 비교, 법령 구조 읽는 법을 보강한다.
- 온라인 부록 QR과 함께 책으로 편집한다.

## 수용 기준

- 홈페이지에서 제품의 정체성이 5초 안에 전달된다.
- 첫 10개 제도 목록이 명확히 보인다.
- 환경영향평가 샘플이 실제 제도 모델의 기준 사례로 작동한다.
- 사용자가 다음 제도 제작 요청을 남길 수 있다.
- 사용자가 출간 알림 또는 문의 의사를 남길 수 있다.
- 하나의 제도 페이지를 만들 때 이 PRD와 한 장 템플릿만 보고도 필요한 구성요소를 빠뜨리지 않는다.

## 리스크와 대응

- `제도`라는 말이 넓고 추상적이다.
  - 대응: 첫 화면에서 환경영향평가 같은 구체 샘플을 바로 보여준다.
- 법령은 계속 바뀐다.
  - 대응: 법령 기준일을 명시하고 홈페이지를 최신성 보완 채널로 둔다.
- 자동 추출 결과가 실제 업무 흐름과 다를 수 있다.
  - 대응: 법령상 구조, 자동 추출, 현장 검증 필요를 구분한다.
- 100개를 한 번에 만들면 범위가 커진다.
  - 대응: 첫 10개 공개 후 제작 요청을 받아 우선순위를 조정한다.
- 책 수요가 불확실하다.
  - 대응: 온라인 반응과 출간 알림을 출판기획서의 근거로 축적한다.
