대상: `docs/institutions-100-manifest.json`에 등록된 1~500번 전체 제도

## 감사 중 수정한 결함

1. 기존 데이터 파일 331개에서 manifest와 달리 빠져 있던 최상위 `category`를 동기화했다.
2. 국가인권위원회 진정에 진정서 작성·제출과 처리 결과 수령·확인 노드를 추가했다. 결과별 통지 경로도 연결했다.
3. 임금체불 진정·지도에 비어 있던 `수사기관` 레인을 실제 송치 사건 수사·기소 노드로 채웠다.
4. 지방공무원 결원보충·승진티오의 승진후보자 레인은 신청인이 아니라 인사발령의 영향을 받는 당사자임을 감사 메타데이터로 명시하고, 임용 결과 통지·확인 행위를 보강했다.
5. 국가사이버안전관리규정 DRF JSON의 중첩 응답(`AdmRulService.조문.조문내용`)을 파서가 읽도록 보완하고 회귀 테스트를 추가했다. 이로써 해당 제도의 자동 검증 불가 8건을 해소했다.
6. 취득세·양도소득세·부동산 거래신고의 단계 역행 후보는 기존 L2 대조에서 정당한 대리 신고·해제 신고 모델링 예외로 확인되어 예외 사유를 구조 데이터에 명시했다. 실제 보완 회귀가 아닌 연결은 강제로 loop로 바꾸지 않았다.

## 남은 검토 순서

1. 지역 조례 예시는 적용 지자체를 하나씩 확정하고 `ordin` 원문 연결로 승격한다.
2. 연도별 편람·지침은 소관 부처의 공식 발행본과 기준연도를 지정한다.
3. L3 medium 152건은 실제 이용자 시나리오 표본을 정해 결과 통지·이의·구제 경로를 수동 확인한다.
4. `outlier`·`no-loop`·`no-sublaw`는 제도별 특성을 확인한 뒤 실제 누락인 경우에만 수정한다.
