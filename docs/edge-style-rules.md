# 업무구조도 연결선·화살표 규칙 v1

기준일: 2026-07-16. 스펙 값의 단일 출처는 `web/src/lib/edge-style.mjs` — 값을 바꾸면 이 문서도 함께 갱신한다.
적용 표면: 데스크톱 보드(DesktopProcessBoard), 모바일 세로 보드(PortraitProcessBoard), 공유용 프로세스 이미지(generate-process-article-image.mjs).

## 1. 레이어 순서
- 연결선·화살촉·라벨은 **항상 노드 카드 뒤**에 깔린다. 카드가 선을 가리는 것은 정상, 선이 카드를 가리는 것은 버그다.
- 구현 주의: 셀 래퍼에 `z-index`를 주면 스태킹 컨텍스트가 생겨 카드가 SVG 아래로 갇힌다. 셀은 `position: relative`만 갖는다.

## 2. 화살촉 (ARROW_HEAD)
- 형태: 꼭지각이 뾰족한 삼각형 `M1,1 L13,5.5 L1,10 Z` (14×11).
- 테두리: 흰색 1.2px — 선·배경·다른 촉과 겹칠 때 형태를 분리한다.
- 색: 아래 유형별 대표 팔레트를 **항상** 쓴다. 선은 표면에 따라 옅은 톤을 쓸 수 있으나 촉은 옅게 하지 않는다(진행 방향이 첫눈에 읽혀야 한다).
- 크기: 보드 14×11(userSpaceOnUse). 고해상 이미지는 1.25배.

## 3. 접촉 (EDGE_END_INSET = 1)
- 경로는 대상 카드 경계 1px 이내에서 끝나고 촉끝이 경계에 닿는다. 촉이 카드에서 떠 보이면 안 된다.

## 4. 유형별 표기
| 유형 | 색 | 대시(보드) | 의미 |
|---|---|---|---|
| sequence | #55685e | 실선 | 순차 진행 |
| message | #0d8a63 | 2 4 | 행위자 간 전달 |
| loop | #2563eb | 5 4 | 회귀·반복 |

## 5. 다중 진입 분산 (FAN_IN_GAP = 12)
- 같은 변으로 여러 선이 들어오면 진입점을 세로로 12px 간격 분산해 촉 겹침을 막는다.

## 6. 라벨
- 라벨은 선과 같은 레이어(노드 뒤)에 있으며 카드에 가려질 수 있다. 라벨이 카드를 덮지 않는다.
