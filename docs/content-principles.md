# 콘텐츠 작성 4대 원칙 (운영자 지시, 2026-07-12)

계약업무의 특성상 아래 네 가지에 따라 **가능한 계약과 절차가 완전히 달라진다**. 제도 콘텐츠를 작성·수정·검증할 때 이 네 가지를 항상 명시하는 것을 최우선 품질 기준으로 삼는다.

| # | 요소 | 콘텐츠에 반영하는 위치 | 예 |
| --- | --- | --- | --- |
| 1 | **누가** (담당자·행위주체) | `submittedDocuments[].actor`, process `lanes`/`actor`, `stakeholders` | 조달업체 / 수요기관 담당자 / 계약담당공무원 / 조달청 |
| 2 | **어떤 서류를** | `submittedDocuments[].documents`, process `output_documents` | 물품구매요청서, 견적서, 이행실적증명, 검사조서 |
| 3 | **언제까지** (법정 기한) | process `deadline`, `applicability`·절차 서술의 기한 문구 | 청구일부터 14일 이내(선금), 5일 이내(대가), 만료 20일 전(갱신) |
| 4 | **어떤 사업종류·금액구간** (적용 대상) | `canvas.applicability` | 종합공사 추정가격 30억원 이상 조달요청 의무, 100억원 미만 적격심사 |

작성 규칙:

1. 새 제도 JSON은 위 4요소가 비어 있으면 완성으로 보지 않는다. 기한과 금액 구간은 근거 조문·기준일과 함께 적는다.
2. 4요소 중 법령 원문으로 확인하지 못한 것은 추측으로 채우지 말고 `fieldVerification`에 올린다.
3. 검증자는 4요소 누락·불일치를 일반 조문 오류와 같은 등급(must-fix)으로 다룬다.
4. 캔버스 구성(2026-07-12 개편): `절차와 법적 근거` / `적용 대상`(구 돈의 흐름) / `제출서류`(담당자별, 구 문서의 흐름) / `병목` / `관련 제도` / `현장 검증 필요`. 목적은 상세 페이지 제목 아래 설명으로 통합했고, 개선점·권한 관계 블록은 화면에서 제거했다(권한 데이터는 비교 화면 `핵심 기관`에서 사용).
5. 분류 체계는 조달 업무의 순차 응집도를 따른다: 등록 → 발주 → 공고 → 입찰 → 심사·평가 → 계약·이행 → 사후관리·제재.
6. 콘텐츠 정렬(priority): 1차 정렬은 절차의 순서(위 분류 순), 2차 정렬은 같은 분류 안에서 **일반적 내용 → 특수한 내용** 순. 예: 발주에서 계약방법 결정(모든 계약에 일반) → 조달요청(중앙조달 특수), 계약·이행에서 선금·대가(일반) → 물가변동 조정(조건부) → 다수공급자계약(조달청 특수).
