## 지문보안토큰 → 신원확인 입찰 전환 (핵심 사실관계)

- 조달청은 나라장터 지문보안토큰 사용의무를 2024-01-01부터 폐지하고, 2024-06 개통한 "차세대 나라장터"에서 스마트폰 기반 신원확인(간편인증·개인용 금융인증서·모바일신분증·차세대 나라장터 앱)으로 전환했다. 증거: 보안뉴스(m.boannews.com/html/detail.html?idx=120087), 정보통신신문(koit.co.kr) 기사, 조달청 차세대 나라장터 FAQ(pps.go.kr fileDown key=202412200023).
- 경과조치: 기존 등록 변경이 없는 입찰자는 종전 지문보안토큰 병행 사용 또는 "지문인식 신원확인 예외신청" 후 사업자용+개인용 인증서 사용 가능. 신규 등록·입찰자 변경 업체는 예외신청 후 사업자용+개인 인증서 사용. 증거: 보안뉴스 기사, just-do-it.co.kr 예외적용 안내.
- 인증서 2종 구분: 사업자용 인증서=업체 등록·전자서명(전자문서 송수신), 개인 인증서/개인인증수단=개인 회원가입·투찰 시 본인확인. 증거: g2b.dima.tools 가이드, keyzard.cc 안내. 유의서 제2조제10·11호·제7조제1항제1호 원문과 일치.

## 확인 못 해 뺀 것 / 조문 번호 없이 쓴 것

- 전자조달법 본문(제7·11·17조) 전문(全文)은 law-cache에 TOC만 있어 조문 존재는 확인했으나 문언 대조는 못 함 → articleVerification에서 uncheckable로 분류, verification.sources가 아닌 unresolved(title-needs-confirmation)로 처리.
- 지문보안토큰 예외신청 서식·처리기한, 차세대 나라장터 개인 신원확인 화면 흐름 등 운영 세부는 조달청 공지·FAQ 근거로만 확인 → fieldVerification 등재.
- 전자입찰서 제출 권장 시각(마감 2시간 전 등)은 실무 안내로만 확인, 법정 기한 아님 → 본문 수치 미기재.
