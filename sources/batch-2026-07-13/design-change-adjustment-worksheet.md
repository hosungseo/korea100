## 확인 못 해 뺀 것 / 조문 번호 없이 쓴 것

- 시행규칙 제74조의2 본문: cache가 TOC라 제명만 확인, 세부 산정식 미확인 → 조문번호는 legalBasis에 병기하되 본문 수치는 시행령·일반조건에서만 인용, articleVerification에서 uncheckable로 분류.
- 물품·용역 계약(물품구매·용역계약일반조건)의 '기타 계약내용 변경(영 제66조)'은 이 제도의 주제(공사 설계변경)와 구분되어 본문에서 제외.
- 표준시장단가 적용 공사의 별도 조정기준(일반조건 제20조제3항)은 cache에 있으나 상세는 fieldVerification로 남김(현장 적용 방식 다양).

## 수정 이력 (2026-07-13, 검증 심사 지적 반영)

검증 심사자 지적 6건을 반영해 JSON(`web/data/institutions/design-change-adjustment.json`)과 본 워크시트를 수정했다.

1. **영 제65조 항 귀속 오류(refuted, must-fix)** — P08·canvas.procedure[5]의 '제65조제1항'을 '제65조'로 낮춰 인용. 근거: law-cache 공사계약일반조건 제21조제4항제1호가 단가 조정 기준으로 '시행령 제65조 제3항'을 인용하며, WebSearch(easylaw·lbox) 재확인 결과 제1항은 원칙 규정이고 증감 공사량 단가 기준은 제3항 계열이다. 시행령 캐시가 TOC뿐이고 협의 규정 항 번호가 버전 간 편차(제3항제3호 vs 제5항)를 보여 항 번호를 확정하지 않고 '제65조'로 인용했다. P06의 제65조제1항(원칙 규정 취지)은 심사자 판단대로 유지.
2. **영 제65조제2항 오귀속(refuted, must-fix)** — P09·canvas.procedure[6]의 '제65조제2항'을 '제65조'로 수정. 협의 결정·불성립시 100분의 50 규정의 정확한 항은 원문 기계 대조 전까지 확정 불가로 판단.
3. **verification 상태·방법·일자(schema-error, must-fix)** — status를 needs-review로, method를 '공개 웹 검색(WebSearch) 교차 대조 — 국가법령정보센터 원문 기계 대조(LAW_OC) 미실시'로, verifiedAt·articleVerification.checkedAt를 2026-07-12로 되돌림.
4. **articleVerification 집계(schema-error, must-fix)** — 시행규칙 제74조의2(2건)와 영 제65조 항 단위 인용(5건: P06·P08·P09·P10·canvas)을 uncheckable로 재분류. citationEntries 23→24, explicitCitationEntries 23→24, articleReferences 32→33(P10에 영 제65조 상위 근거 추가분), verified 32→26, missing 0, uncheckable 0→7. 합계 26+0+7=33 일치. needs-review 게이트((missing+uncheckable)≥1) 충족.
5. **P04 현장확인 주체(wording, should-fix)** — 인용 조문(일반조건 제19조의3제2항·제19조의2제2항)의 법적 주체는 계약담당공무원이므로 P04.action에 '법적 주체는 계약담당공무원, 실무상 공사감독관이 현장확인 수행' 취지를 명시하고 fieldVerification에 감독관 대행 관행 항목을 추가. lane·actor(공사감독관)는 실무 서술로 유지.
6. **P10 상위 근거 병기(wording, should-fix)** — 저가낙찰 대규모 증액 심의·승인의 상위 근거로 P10.legal_basis와 canvas.authorities[2]에 영 제65조를 병기(관리자 기계 대조 전이라 항 번호 없이 '제65조'). 일반조건 제20조제6항은 유지.
