# 변경 이력 (CHANGELOG)

이 사이트의 모든 공개 변경 사항을 `X.Y.Z` 체계로 기록합니다.

- **X (major)**: 데이터 계약·사이트 구조가 바뀌어 기존 링크나 데이터 해석이 달라지는 개편
- **Y (minor)**: 제도 추가, 새 화면·기능, 콘텐츠 확충
- **Z (patch)**: 오탈자·조문 정정, 버그 수정, 문구·스타일 조정

버전은 `web/package.json`의 `version`과 사이트 푸터에 함께 표시하고, main 머지 시 git 태그(`vX.Y.Z`)를 남깁니다.

## v0.1.0 — 2026-07-12

첫 관리 버전. 원본인 **[한 장으로 끝내는 대한민국 제도 100(korea100)](https://github.com/hosungseo/korea100)** 을 포크해 공공조달 도메인으로 전환한 시점부터의 차이를 기록한다.

### 제도100(korea100) 대비 달라진 점

**콘텐츠**
- 대상 도메인 교체: 일반 행정제도 101개 → **공공조달·계약 제도 100개 목표**. 현재 3개 공개(조달업체 등록, 수의계약, 부정당업자 입찰참가자격 제한). korea100 원본 콘텐츠는 `legacy-korea100` 브랜치에 보존.
- 분야 체계 교체: 국정 전반 10개 분야 → 조달 업무 흐름 기준 분야(조달시장 진입과 참여 준비, 계약방법 결정과 입찰 절차, 제재·분쟁·권리구제 등).
- 근거 법령 축 교체: 국가계약법·시행령·시행규칙, 조달사업법, 계약예규, 나라장터 등록규정 등 조달 법령 중심.

**검증**
- 조문 검증 파이프라인을 국가법령정보센터 Open API 기반 **자동 대조**로 운용하고, 화면 표기를 '검증 완료' 대신 **'자동대조 완료'** 로 명시(사람 심사로 오해 방지).
- 노드 단위 검증 표시(VerificationMark) 추가, 현장 검증 대장 화면 정비.

**화면**
- 업무구조도에서 **진행률 어휘(완료·현재·대기) 제거**: 정적 예시 화면이 이용자 자신의 진행 상태로 오해되는 문제를 없애고, 핵심 단계·병목·보완 회귀만 강조하는 표기로 전환. 상태 어휘는 `web/src/lib/node-status.ts` 단일 출처로 통합.
- '같은 기준으로 비교'를 구조 지표(노드·레인·게이트 수) 나열에서 **제도 실질 비교**(무엇을 하는 제도인가, 누가 관여하나, 결정 권한, 근거 법령, 실무에서 걸리는 곳)로 개편.
- 메인 카탈로그 단순화: 검색창 placeholder '제도', 왼쪽 레일을 주제별 바로가기 전용으로, 정렬 토글·중복 필터·분야/노드 열 제거.
- 상단 메뉴 축소(제도 대장·요청하기), 현장 검증 대장은 푸터로 이동.

**운영**
